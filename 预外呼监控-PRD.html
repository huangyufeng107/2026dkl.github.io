1. 预外呼监控

F1 数据概览

2）交互说明

页面入口：系统运维 > 系统监控 > 预外呼监控，默认进入"数据概览"Tab。用户无需筛选即可查看当前周期全量指标。点击任意指标卡片弹出指标说明浮层（metricInfoPopover），展示指标定义与计算口径。同步失败量、话单超时未回传、录音丢失量、线路拒呼、线路未呼卡片值大于 0 时高亮显示警告样式（metric-alert），用于提示运维关注。

3）数据来源

后端接口：预外呼指标概览查询接口（GET /api/ops/predialer/metrics），参数为 startDate、endDate、可选 attentionFocus。指标计算在后端完成，前端仅渲染。

4）逻辑说明

指标分为两组：

  • 任务流转与坐席分配（8 个指标）：预外呼任务总量、已推送任务量、同步失败量、已分配坐席（总量）、已分配坐席（通话状态=坐席接听）、线索下发量、线索下发量（通话状态=坐席接听）、线索下发量（通话状态≠坐席接听）。
  • 外呼与回传监控（15 个概览指标）：任务处理中、任务已处理、有效任务量、话单已回传、话单待回传、话单超时未回传、录音丢失量、外呼接通量、有效通话量、首呼接通率、2~N呼接通率、线路拒呼、线路未呼、呼叫接通时长、每通呼叫接通时长。同步批次汇总另展示“日人均承载量”。

核心计算口径：推送率 = 已推送任务量 ÷ 预外呼任务总量；处理率 = 任务已处理 ÷ 已推送任务量；接通率 = 外呼接通量 ÷ 有效任务量；有效通话率 = 有效通话量 ÷ 外呼接通量；话单已回传 / 话单待回传 / 话单超时未回传均按话单统计，不按任务去重；外呼接通量按接通话单统计，不按任务去重；有效通话量 = 建联成功且通话时长 ≥ 30秒的话单数；有效任务量 = 已推送任务量 -（线路拒呼且外呼已完成任务 + 线路未呼且外呼已完成任务 + 存在待回传话单的任务 + 任务中止 + 用户黑名单）；已分配坐席按坐席ID去重统计；日人均承载量 = 有效任务量 ÷ 去重坐席人数。

F2 明细列表

2）交互说明

页面入口：系统运维 > 系统监控 > 预外呼监控 > 明细列表 Tab。进入 Tab 后顶部显示筛选面板（同步日期、任务系统状态、任务异常原因、预外呼任务状态、最新外呼结果），筛选面板仅在此 Tab 及同步批次汇总 Tab 显示。点击"查询"按钮按筛选条件刷新列表；点击"重置"清空所有筛选条件并刷新；点击"刷新"重新加载当前筛选结果。列表支持分页（5/10/20/50条/页）和页码跳转。点击"查看"按钮打开任务详情弹窗，展示任务基本信息和外呼话单回传列表。点击"导出数据"按钮下载 Excel 文件。

3）数据来源

后端接口：预外呼任务列表查询接口（GET /api/ops/predialer/tasks），参数含 startDate、endDate、status、systemStatus、exceptionReason、callResult、keyword、attentionFocus、page、pageSize。

4）逻辑说明

列表表头 14 列：同步日期、人工任务编码、预外呼任务ID、总部线索ID、客户（含手机号）、意向车系、任务系统状态、任务异常原因、外呼次数（多次时显示"多次"警告标签）、预外呼任务状态（处理中超时显示"处理中（超时）"标签）、最新外呼结果、更新时间、已下发线索（是/否）、操作（查看按钮）。筛选逻辑：日期范围两端取包含；任务系统状态精确匹配；异常原因精确匹配；预外呼任务状态精确匹配；最新外呼结果按最新一条话单结果匹配；关键词在任务ID、线索ID、客户名、手机号中模糊匹配；attentionFocus 定位筛选逻辑：sync-failed 匹配同步失败任务，timeout-pending 匹配超时未回传任务，recording-missing 匹配录音丢失任务。

2）交互说明

4）逻辑说明

2）交互说明

  • 预警事件中心：表格展示预警编号、等级（P1/P2 标签）、预警类型、关联任务、自动处置、责任人、状态、操作（确认/关闭按钮）。
  • 预警规则与通知升级：表格展示规则名称、触发条件、等级、通知对象、未确认升级、重复告警策略、状态。右上角"规则配置"按钮打开预警规则配置弹窗。

预警规则配置弹窗包含三个规则分组：

  • 话单回传超时：超时阈值（小时，默认24）、预警等级（P1/P2）、通知对象（账号人员多选）、未确认升级（分钟，默认5，升级至运维主管）。
  • 同步失败率异常：统计窗口（分钟，默认15）、失败率阈值（%，默认5）、预警等级（P2默认）、通知对象、未确认升级（分钟，默认30，升级至业务负责人）。
  • 录音缺失：触发条件（已回传话单无录音）、预警等级（P2默认）、通知对象、未确认升级（分钟，默认60，创建人工工单）。
  • 重复告警抑制：同一任务合并窗口（分钟，默认30）。

3）数据来源

预警事件：后端接口（GET /api/ops/predialer/alerts）。预警规则：后端接口（GET /api/ops/predialer/alert-rules）。通知对象选择器：复用账号人员列表（reviewUserOptions），包含账号、姓名、角色、部门。

4）逻辑说明

预警状态流转：待确认 → 处理中（点击确认）→ 已关闭（点击关闭）。确认操作将 owner 变更为"DCC运营"。通知对象选择器支持多选、全选、清除所选，选择结果存储为账号ID数组。
