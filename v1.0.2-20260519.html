created_atDATETIME-创建时间，系统自动填充
updated_atDATETIME-更新时间，系统自动维护

接口列表
功能方法路径说明
品牌列表（分页）GET/api/v1/brands支持按名称/编码/状态筛选
新增品牌POST/api/v1/brands-
编辑品牌PUT/api/v1/brands/{id}brand_code 不可修改
启用/停用PATCH/api/v1/brands/{id}/status联动停用下属车系/门店
删除品牌DELETE/api/v1/brands/{id}有关联数据时禁止删除
业务规则
  • 🔒 brand_code 全局唯一，创建后不可修改（与主机厂系统绑定）
  • ⚠️ 停用品牌时，系统向前端展示警告："该品牌下共有 N 个车系、M 个门店，停用将同步停用，是否确认？"
  • 🚫 删除前校验：该品牌下存在关联数据（车系/门店/线索）时不允许删除，返回关联数量提示
  • 📶 列表默认按 sort_order 升序排列，sort_order 相同时按 created_at 降序

字段名类型必填说明

api_secretVARCHAR(200)○API Secret（仅生成时展示一次，加密存储）
api_key_expireDATETIME○Key 过期时间，NULL = 永不过期
brand_idsJSON○授权品牌 ID 列表，NULL = 全部品牌
store_idsJSON○授权门店 ID 列表，NULL = 全部门店

账号类型品牌范围门店范围API 推送
平台方（type=1）全部品牌全部门店全品牌推送
媒体方（type=2）授权品牌列表授权品牌下所有门店仅授权品牌

🔴 安全要求：API Secret 采用单向加密存储（如 bcrypt），仅在生成时明文展示一次。重置 API Key 后，旧 Key 立即失效（Redis 黑名单，5分钟TTL）。停用账号后，该账号 API 调用立即返回 401。

🔌 线索推送 OpenAPI
标准化接口，供外部供应商推送线索数据，支持 HMAC-SHA256 签名认证
认证方式
HMAC-SHA256 签名认证

请求头携带以下字段：

X-Api-Key: {api_key} X-Timestamp: {unix_timestamp_seconds} // 服务器时间±5分钟内有效 X-Nonce: {6位随机字符串} X-Signature: {hmac_sha256_signature} Content-Type: application/json

签名算法：

// 1. 构造签名串 sign_str = api_key + "\n" + timestamp + "\n" + nonce + "\n" + md5(request_body) // 2. 计算签名 X-Signature = HmacSHA256(sign_str, api_secret) // 防重放：nonce + timestamp 组合 10 分钟内不允许重复（Redis 验证）
推送接口
POST /open/v1/leads/push
{ "source_lead_id": "媒体平台侧线索唯一ID", // 必填，用于去重 "source_channel": "dongchedi_form", // 必填，渠道标识 "push_time": "2026-05-14T10:00:00+08:00", "customer": { "name": "张三", // 必填 "phone": "Base64(AES加密后手机号)", // 必填，加密传输 "gender": 1, // 1=男 2=女 0=未知 "province": "广东省", "city": "深圳市", "district": "南山区" }, "intent": { "brand_code": "LIXIANG", // 必填，与系统品牌编码对齐 "series_code": "LIXIANG_L9", // 可选 "dealer_code": "DLR_SZ_001", // 可选，直推到门店 "budget_min": 30, "budget_max": 50, "purchase_timeline": 2, // 1=一月内 2=一至三月 3=三月以上 "lead_level": "A" // A/B/C/D }, "source_info": { "utm_source": "dongchedi", "utm_medium": "cpc", "utm_campaign": "lixiang_l9_2026q2", "clue_type": "试驾" }, "custom_fields": { // 与系统自定义字段 field_key 对应 "marital_status": "1", "referral_source": "朋友推荐" } }

1006门店编码不存在或已停用400
1007手机号格式错误（解密后校验）400
1008超出每日推送配额429

手机号加密规范
// 加密算法：AES-128-CBC + PKCS7 Padding Key = api_secret 前 16 字符（ASCII） IV = X-Nonce（6位）+ account_code 前 10 字符（共16位） encrypted_phone = AES_CBC_Encrypt(phone, key, iv) X-Phone = Base64_Encode(encrypted_phone)
去重规则 （暂不实现）

满足以下任意条件视为重复线索：

  • ① 同一供应商 + 相同 source_lead_id
  • ② 同一手机号 + 同一品牌 + 72小时内已存在（时间窗口可后台配置）

重复策略（可后台配置）：

策略行为
忽略（默认）返回 409 Conflict，不写入
更新用新数据覆盖原线索部分字段，返回原 lead_id
追加新建线索并打标"疑似重复"，等待人工处理

📝 推送日志
记录所有 OpenAPI 推送请求的完整日志，支持失败重推和问题排查
日志字段
字段说明
supplier_id推送方供应商 ID
request_ip推送方 IP 地址
request_header请求头 JSON（含签名信息，脱敏 Secret）
request_body原始请求体（手机号已加密，保留原文）
response_codeHTTP 响应状态码
response_body系统响应内容
error_message失败时的错误说明
lead_id成功时关联的线索 ID
is_success1=成功 0=失败
💡 推送日志保留 180 天，过期自动归档至冷存储。失败日志支持"失败重推"功能，重新触发推送并创建新的日志记录。

🔐 权限设计
基于角色的权限隔离，确保多租户数据安全
角色权限矩阵
角色基础数据线索查看供应商管理系统配置
总部超级管理员✅ 全部✅ 全部✅✅
总部运营人员✅ 所属品牌✅ 所属品牌❌❌
供应商管理员❌✅ 仅自己推送的❌❌
经销商运营人员❌✅ 仅本门店❌❌
数据隔离规则
  • 🔒 所有列表接口自动附加数据范围过滤（WHERE brand_id IN (...) / store_id IN (...)）
  • 🔒 供应商账号调用 OpenAPI 时，系统自动校验推送品牌是否在授权范围内
  • 🔒 经销商运营人员无法查看其他门店线索，无法修改线索的品牌/来源信息
  • 🔒 导出接口同样受数据范围限制，不可导出权限外数据
  • 🔒 敏感操作（删除、导出、重置密钥）记录操作日志，保留 365 天
