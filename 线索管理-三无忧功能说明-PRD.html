四. 功能说明

三无忧活动看板

1）活动概览与当日指标

页面入口线索管理 > 三无忧。

适用角色具有“三无忧查看”权限的运营、DCC 管理员；无权限时不展示三级菜单，直接访问提示“暂无访问权限”。

操作说明进入页面自动加载活动状态、准入车系、每日上限、外呼开始时间、去重窗口，以及今日候选、去重过滤、短信成功率、DCC 培育任务、滚存待推送五项指标。

数据来源与取值逻辑活动配置表、当日批次统计表和任务回收统计。今日候选为准入和车系过滤后的数量；去重过滤为建联、黑名单、同号任一命中数量；短信成功率=短信成功数/短信发送数×100%，分母为 0 时显示“—”；滚存待推送为候选去重后未进入当日推送配额的数量。

状态与验收加载中展示骨架；接口空数据展示 0 或“—”；网络异常展示重试入口。当活动停用时，应展示“停用”并禁用配置保存及导出以外的活动执行操作。

规则配置

1）展开、编辑与保存基础规则

页面入口活动看板 > 规则配置 > 展开配置。

适用角色查看权限可查看；“三无忧配置”权限可编辑与保存；仅查看用户的字段、增删按钮与保存按钮均禁用。

操作说明点击“展开配置/收起配置”切换配置区；选择准入意向车系（支持全选、清空）；填写每日推送上限、AI/人工去重窗口、短信模板、推送外呼时间、禁止推送时间、同号窗口，点击保存。

字段/参数每日推送上限：正整数；准入意向车系：至少选 1 项；去重窗口及同号窗口：正整数天，前端展示 T-N；短信模板：下拉单选；推送及禁止推送时间：HH:mm - HH:mm 区间。原型默认：10000、N6/N7/NX8、30 天、冷线索-三无忧、09:30-18:00、12:00-14:00 / 20:00-09:00。

数据变更更新活动配置及其版本、更新人、更新时间；批次明细和导出必须保存执行时使用的配置快照，不因后续配置修改而回溯改变。

异常处理与验收缺失字段时阻止保存、聚焦并提示具体字段；当未选择车系时，应提示“请选择准入意向车系”；当保存时版本与服务端版本不一致，应提示并要求刷新后重试；成功后应刷新摘要区。

2）维护分层标签与车系话术映射

页面入口展开配置 > 分层标签映射权重 / 意向车系映射话术任务。

操作说明可新增或删除规则行；分层选择 P1-P5 后自动带出不可编辑标签映射；填写权重。车系话术表按 N6、N7、NX8 选择车系，填写一知任务 ID 和话术任务。

数据来源与取值逻辑P1=历史试驾过/历史到店过，P2=门店跟进次数≥3次，P3=休眠未购，P4=休眠失联，P5=历史剩余数据。候选线索按命中层级取得权重，权重由大到小排序；同权重时【待确认：按最近跟进时间或线索创建时间排序】。车系须匹配对应一知任务 ID 后才可推送。

字段规则每张表至少保留 1 行；权重为 1-999 的整数；同一分层和同一车系不得重复；一知任务 ID、话术任务均不能为空。删除当前行仅影响未保存的编辑态，保存后生效。

验收标准当用户选择 P2 时，标签映射应自动显示“门店跟进次数 ≥ 3 次”；当映射行为空或重复时，应阻止保存并指出行号；当车系不存在有效话术映射时，该车系线索不得进入一知推送。

批次记录与详情

1）批次记录分页、日期筛选与查看

页面入口三无忧活动看板 > 批次记录。

操作说明查看批次日期、候选量、去重过滤、短信成功、DCC 培育任务、剩余量和批次状态；可切换每页 5/10/20/50 条、翻页及点击查看。

数据来源与逻辑批次以业务日期为粒度；状态枚举为“执行中”“已完成”“失败”。执行中批次实时或定时刷新统计，已完成批次为固定快照。无记录时展示“暂无批次记录”。

验收标准当页码超出总页数时，应自动回到最后一页；当点击不存在批次时，应提示“未找到该批次详情”；列表加载失败不应显示旧页码造成误导。

2）批次详情：执行概览、批量明细、执行日志

页面入口批次记录 > 查看。

操作说明默认打开执行概览；可切换“批量明细”“执行日志”；点击返回列表回到批次记录。批量明细支持分页与横向查看全部字段。

字段/参数概览展示批次日期、候选量、去重过滤、短信成功、DCC 培育任务、系统异常原因量、剩余量、批次状态；分层表与车系表展示候选、去重、短信、任务、异常、剩余；明细包含批次号、线索/任务编码、客户脱敏手机号、车系、门店、分层权重、短信/系统状态及运行配置快照。

数据来源与取值逻辑分层表按 P1→P5；车系表按配置车系；系统异常原因量优先取批次汇总字段，缺失时汇总各分层异常量。执行日志按发生时间升序，节点包括候选池生成、短信预触达、推送一知、结果回收。

验收标准当批次状态为执行中时，应展示当前已完成节点和“执行中”；当明细为空时，应展示空状态且不报错；当系统状态为失败时，应展示系统异常原因。

数据导出

1）按日期范围导出三无忧明细

页面入口批次记录右上角 > 开始日期、结束日期 > 导出明细。

适用角色具有“三无忧导出”权限；无权限时隐藏导出按钮。

操作说明输入日期范围后点击导出，系统按批次日期过滤；导出内容为所有匹配批次的明细和每条明细关联的配置快照。

数据格式CSV，UTF-8 with BOM；文件名：三无忧全部数据_{开始日期}_{结束日期}.csv。字段包括批次号、线索及任务信息、客户信息、分层、短信/系统状态、每日上限、车系、去重窗口、短信模板、时间窗、映射规则。

异常处理与验收开始日期晚于结束日期时阻止导出并提示“开始日期不能晚于结束日期”；范围无数据时提示“当前日期范围内暂无可导出数据”；当导出成功时，文件应可被 Excel 正常打开且中文不乱码。

待确认项

  1. 8 个建议接口的实际网关路径、鉴权方式和统一响应结构（影响前后端联调与测试）。
  2. 候选线索同权重时的排序规则，以及跨层命中时是否取最高权重或允许重复（影响后端筛选、批次统计、测试验收）。
  3. 短信失败的重试策略、超时阈值和失败终态（影响任务调度与运营指标）。
  4. 一知推送及结果回收的幂等键、回调时限、补偿/重跑机制（影响后端可靠性和异常处理）。
  5. 导出日期范围上限、异步导出阈值和脱敏策略（影响性能、权限和合规）。
